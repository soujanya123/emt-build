The RealChat public chat service includes a chat server for handling chat requests, and a basic web server for serving the chat client files. To allow access for users behind firewalls, both servers share a single address.

The Control Center service, this application, runs on a separate address.

User Permissions
RealChat supports three levels of user permissions. These include normal users, operators, and administrators. For more details about the hierarchy please visit: Chat Room Administration.

When people first join a chat, they are logged in as a normal user without any permissions. To become an administrator or operator, they can use the /pass command. [shared passwords option]

When integrating with a membership database, you can define lists of user names that belong to the operators or administrators group. When a user logs in, if a name is matched to the list, the user is automatically promoted to an operator or administrator role. [predefined lists option]

Log Settings
RealChat log files are stored in the /logs subdirectory of the directory where you installed RealChat. Chat transcripts are grouped by virtual server and files are named like this: NN-chat.log, where NN is the ID of the virtual server.

Client Profiles
Chat client profiles are predefined sets of user permissions and interface settings. You can define up to five profiles here, each suitable for a different user group. For example, you may have one profile for paying customers, with all the features enabled, and also have a restricted profile for free or guest members.

HMAC authentication should be turned off, for the preview to work.
